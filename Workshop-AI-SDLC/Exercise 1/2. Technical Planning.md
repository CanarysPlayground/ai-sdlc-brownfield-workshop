#### Step 2.2 /constitution ‚Äî Update Project Constitution

Spec Kit tutorials explicitly show that /constitution updates the project constitution, and teams can extend it using custom files such as agents.md. 

This file tells Copilot/Spec Kit how your application must be built.

**Prompt 1: Update Constitution**
```/constitution
Update the project constitution to include the design guardrails from .github/agents.md for the OctoCAT Supply Chain System.
```

#### Step 2.2  /specify ‚Äî Generate Feature Specification

The /specify command creates the high-level feature specification (what/why), based on your GitHub Issue and context.

**Prompt 1: Generate Feature Specification**
```
/speckit.specify 

Feature: Cart Icon with Item Count Badge

Use these context files:
- docs/github-issue-draft.md
- .github/agents.md

Produce:
- User stories
- Acceptance criteria
- Non-functional requirements
- API expectations
- Accessibility & dark mode requirements
Follow project constitution rules.
```

####  Step 2.3 ‚Äî Optional: /clarify Loop
Use if Spec Kit identifies ambiguities.

**Prompt 2: Clarify Ambiguities**
```/speckit.clarify

Find unresolved questions in the cart icon spec and propose options +
recommended defaults following agents.md.
```

#### Step 2.4 ‚Äî /plan ‚Äî Technical Architecture Plan
The /plan command converts the spec into a technical plan.
(Supported in Spec Kit Quick Start: /speckit.plan creates technical architecture.)

**Prompt 3: Generate Technical Architecture Plan**
```
/speckit.plan

Generate a full-stack technical plan based on the spec + agents.md:

Include:
- Frontend files to create (types, context, CartIcon)
- Backend migration, repository, service, routes
- API contract shapes
- localStorage sync rules
- Testing strategy
- Milestones (Backend ‚Üí Frontend ‚Üí Integration ‚Üí Tests)

Follow repository structure exactly.
```

#### Step 2.5 ‚Äî /tasks ‚Äî Produce Implementation Tasks

The /tasks command converts the technical plan into small, executable tasks.
**Prompt 4: Generate Implementation Tasks**
```
/speckit.tasks

Convert the technical plan into implementation tasks.

Requirements:
- Group by milestones
- Each task includes file path, expected output, test requirement
- Tasks should be 15‚Äì45 minutes each
- Add final ‚ÄúDefinition of Done‚Äù

Follow agents.md guardrails.
```

#### Step 2.6 (Optional) /analyze ‚Äî Quality Check
Use /analyze to validate the spec/plan/tasks.

**Prompt 5: Quality Check**
```/speckit.analyze

Evaluate constitution, spec, plan, tasks for:

- Alignment with agents.md
- Missing architecture decisions
- Risk areas
- Testability
- Completeness for implementation

Return: Ready, Warnings, Blocking Items.
```
