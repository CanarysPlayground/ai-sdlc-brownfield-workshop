### **STEP 1: Setup Figma MCP Server** (10 minutes)

#### 1.1 Create MCP Configuration File
- Navigate to `.vscode/` folder in your project
- Create a new file named `mcp.json` (if it doesn't exist)

#### 1.2 Add Figma MCP Configuration
Paste the following configuration:

```json
{
  "mcpServers": {
    "figma": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-figma"
      ],
      "env": {
        "FIGMA_PERSONAL_ACCESS_TOKEN": ""
      }
    }
  }
}
```

#### 1.3 Get Figma Personal Access Token
- Open Figma in your browser
- Go to **Settings** â†’ **Account** â†’ **Personal Access Tokens**
- Click **Generate new token**
- Give it a name (e.g., "Workshop Token")
- Copy the token

#### 1.4 Add Token to Configuration
- Paste your token in the `FIGMA_PERSONAL_ACCESS_TOKEN` field in `mcp.json`
- Save the file

#### 1.5 Start Figma MCP Server
- Press `Cmd/Ctrl + Shift + P` in VS Code
- Type: `MCP: List servers`
- Select `figma` â†’ `Start server`
- Verify the server status shows "Running"

### **STEP 7: Create New Component Structure** (10 minutes)

#### 7.1 Create Components Folder
```bash
mkdir -p frontend/src/components/products
```

#### 7.2 Plan Component Architecture
Review your TRD to understand the component structure you'll create:
- ProductCard
- ProductGrid
- ProductFilters
- ProductSearch
- ProductSort
- ProductDetailModal

#### 7.3 Switch Copilot to Agent Mode
- Open Copilot Chat
- Switch mode from "Chat" to "Agent"

#### 7.4 Generate Component Structure Prompt
Paste this comprehensive prompt:

```
@workspace Based on the specifications in docs/specifications/, refactor the Product 
page with the following new component structure:

1. Create ProductGrid component (frontend/src/components/products/ProductGrid.tsx)
2. Create ProductCard component (frontend/src/components/products/ProductCard.tsx)
3. Create ProductFilters component (frontend/src/components/products/ProductFilters.tsx)
4. Create ProductSearch component (frontend/src/components/products/ProductSearch.tsx)
5. Create ProductSort component (frontend/src/components/products/ProductSort.tsx)
6. Refactor Products.tsx to use these new components

Follow the design specifications in product-page-design-spec.md for:
- Color scheme
- Typography
- Spacing
- Component dimensions
- Interactive states

Implement:
- Responsive design (mobile-first)
- Accessibility (ARIA labels, keyboard navigation)
- Loading states
- Error handling
- Empty states
- Smooth transitions

Use Tailwind CSS for styling and maintain dark mode support.
```

#### 7.5 Monitor Agent Progress
- Watch as Agent analyzes the specifications
- Review the implementation plan
- Approve or adjust as needed
